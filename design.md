# 字体排版 Design · 「我的清单」

> 基于 `index.html`(4166 行,内嵌 CSS)的全量扫描,2026-07。
> 目标:参照「系统原生字体栈 + 8 级字号阶梯」的排版方案,盘点现状、给出迁移路径。
>
> **状态:✅ 已实施**。token 已加入 `:root`,全站 175 处 font-size 中的文本类已全部迁移到 `--fs-*`,
> `--round` 已删除,行高收敛为三档 token,letter-spacing 统一为 em。
> 星标 / emoji / 箭头等字形尺寸(20/22/26/34px 等)按方案保留组件内定义。
> 注:logo 实际生效字号原为 19px(被「视觉覆盖层」全局覆写),迁移后归入 18px Title 级。

采用系统原生字体栈(SF Pro / PingFang SC),无需加载外部字体,首屏渲染零延迟——这一点已经和参考方案一致,**保留**。

| 变量 | 值 | 用途 | 使用次数 | 状态 |
|---|---|---|---|---|
| `--zh` | `-apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif` | 中文 / 界面主字体 | 42 | ✅ 保留 |
| `--en` | `-apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif` | 数字 / 英文(统计数、年份、计数徽章) | 26 | ✅ 保留,中英分栈是好做法 |
| `--round` | 与 `--zh` 完全相同 | 标题 / 数字(历史遗留) | 21 | ⚠️ 冗余,建议删除并替换为 `--zh` |

## 03 需要修改的 7 个问题

1. **零字号 token** —— 15 档字号全部硬编码,改字号要全局搜索替换。
2. **层级过密** —— 17 / 18 / 19 / 20px 相邻 1px 四档,肉眼无法区分层级。
3. **过小字号** —— 9px(1 处)、10px(10 处)低于可读下限,统一提升到 11px。
4. **标题天花板低** —— 最大标题 19px,与 14px 正文对比不足,页面缺少视觉锚点。
5. **行高散乱** —— 9 种值收敛为三档:`1`(紧凑数字/按钮)、`1.3`(标题)、`1.6`(正文)。
6. **冗余变量** —— `--round` 与 `--zh` 完全相同,删除。
7. **字距单位混用** —— letter-spacing 同时存在 em 与 px(`.08em` 与 `2px` 并存),统一为 em。

## 已经做对的,保留

- ✅ 系统原生字体栈,零外部字体加载(与参考方案一致)
- ✅ 中英文分栈:数字、年份、计数走 `--en`
- ✅ 手机端输入框 16px 防 iOS 聚焦自动放大,44px 触控高度达标
- ✅ 层级靠字重(600/700 主力),密集界面里比放大字号更克制
- ✅ 正文行高 1.6,中文正文的合理值

## 04 建议的 8 级字号阶梯

沿用参考方案的 8 级结构,但按这个**密集工具型应用**的密度整体下调一档(参考图 36→12 适合展示型页面,清单工具用 28→11 更合身)。

| 级别 | 名称 | 字号 / 字重 / 行高 | 对应现有元素 |
|---|---|---|---|
| 1 | 展示标题 Display | **28px** / 700 / 1.2 | 空状态大标题、欢迎场景(新增层级) |
| 2 | 页面标题 Heading | **22px** / 700 / 1.25 | `.vtitle` 视图标题(由 19px 升级) |
| 3 | 大标题 Title | **18px** / 700 / 1.3 | `.snum` 统计数字、手机端页面标题(合并 17–20px) |
| 4 | 标题 Headline | **16px** / 600 / 1.4 | `.mtitle` 弹窗标题、笔记标题、手机输入框 |
| 5 | 小标题 Subheadline | **14px** / 600 / 1.45 | `.cname` 条目名称、表头强调(与正文同号,靠字重区分) |
| 6 | 正文内容 Body | **14px** / 400 / 1.6 | `.cnote` 备注、表单输入、body 基准(手机端 15px) |
| 7 | 辅助文字 Caption | **13px** / 400–600 / 1.5 | 按钮、导航项、徽章、meta(合并 12px 与 13px) |
| 8 | 标注微缩 Footnote | **11px** / 500–600 / 1.4 | `.nlbl` 节标签、`.slbl`、计数(全站最小下限) |

### Token 定义(建议加入 `:root`)

```css
:root{
  /* === 字号阶梯 === */
  --fs-display:28px;   /* 展示标题 */
  --fs-heading:22px;   /* 页面标题 */
  --fs-title:18px;     /* 大标题/统计数字 */
  --fs-headline:16px;  /* 弹窗标题/笔记标题 */
  --fs-sub:14px;       /* 小标题(600 字重) */
  --fs-body:14px;      /* 正文基准 */
  --fs-caption:13px;   /* 按钮/徽章/meta */
  --fs-foot:11px;      /* 节标签/计数,最小下限 */
  /* === 行高三档 === */
  --lh-tight:1.3;      /* 标题 */
  --lh-normal:1.45;    /* 小标题/紧凑正文 */
  --lh-loose:1.6;      /* 正文 */
}
@media (max-width:768px){
  :root{ --fs-body:15px; }  /* 手机端正文升半档;--fs-headline 16px 防缩放不动 */
}
```

## 05 迁移映射

绝大多数是「向最近一级取整」,视觉变化极小;真正能看出变化的只有两处:**页面标题变大**(19→22)、**节标签变大**(10→11)。

| 现有字号 | 迁移到 | 说明 |
|---|---|---|
| ~~9px~~ ~~10px~~ | 11px Footnote | 11 处,提升可读性;节标签保留大写 + `.08em` 字距 |
| 11px | 11px Footnote | 不变;个别 meta 场景可视情况升 13px |
| 12px · 13px | 13px Caption | 68 处主力合并,信息密度几乎不变 |
| 14px · 15px | 14px Body / Sub | 桌面 14、手机基准 15 保留;600 字重即 Subheadline |
| 16px · 17px | 16px Headline | 手机输入框 16px 防缩放特性顺带满足 |
| 18px–20px | 18px Title | 四档并一档,统计数字、手机标题归口 |
| 19px(`.vtitle`) | 22px Heading | 唯一「升级」项,建立页面锚点 |
| 22 · 26 · 34px | 组件内定义 | 星标 / emoji 属图形尺寸,不进文字阶梯 |

### 实施顺序建议

1. `:root` 加入 token(纯新增,零风险)。
2. 删除 `--round`,21 处引用替换为 `--zh`。
3. 9px / 10px → `var(--fs-foot)`(11 处,可读性收益最大)。
4. 12px / 13px → `var(--fs-caption)`(68 处,机械替换)。
5. 其余字号按映射表逐档替换为 token。
6. 行高收敛:1.55 / 1.65 / 1.75 等散值归入三档。
7. letter-spacing 统一 em(`2px` → `.15em` 等价换算)。

# 间距系统 Spacing

> 基于 `index.html` 全站 padding / margin / gap 扫描(441 处取值),2026-07。
> **状态:✅ 已实施**。`--sp-*` 12 级 token 已加入 `:root`,418 处间距值完成迁移
> (奇数 +1 取整、18/22/26/28 归入 20/24、偶数同值替换);
> `calc()` 表达式与功能常量(34/36/38/56/60/124px)按方案保留原样。

## 06 间距现状

- **零间距 token**,所有值硬编码,共 **29 档**(0–124px)。
- 高频主力全部在偶数网格上:10px(51 处)、8px(47)、6px(47)、12px(41)、14px(34)、4px(30)、2px(25)。
- **奇数值 1 / 3 / 5 / 7 / 9 / 11 / 13 / 15px 共 87 处(约 20%)**,不在任何网格上,是节奏被打乱的主因。
- 18px(15 处)、22px(11 处)、26 / 28px 偏离 4px 主节奏。
- 组件功能常量(不算间距):34 / 36 / 38px 为输入框给搜索图标、下拉箭头留的避让宽度;60px 时间线缩进;124px 手机底部栏让位;56px 空状态场景留白。

## 07 建议的 12 级间距阶梯

2px 基准、4px 主节奏、大间距 8px 节奏。参考方案为 4–80px;本应用信息密度高(2 / 6 / 10 / 14px 是主力),故保留 2px 半档、上限收至 40px,同为 12 级:

| token | 值 | 典型场景 |
|---|---|---|
| `--sp-2` | 2px | 徽章内缝、微调 |
| `--sp-4` | 4px | 标签间隙、图标与文字间距 |
| `--sp-6` | 6px | 按钮组 gap、卡片内小间距 |
| `--sp-8` | 8px | 控件内边距、列表项 gap |
| `--sp-10` | 10px | 输入框/导航项内边距 |
| `--sp-12` | 12px | 卡片内边距、区块小间距 |
| `--sp-14` | 14px | 手机端内容边距 |
| `--sp-16` | 16px | 按钮横向内边距 |
| `--sp-20` | 20px | 区块间距 |
| `--sp-24` | 24px | 内容区/弹窗边距 |
| `--sp-32` | 32px | 大区块分隔 |
| `--sp-40` | 40px | 场景级留白 |

```css
:root{
  /* === 12 级间距阶梯(按值命名) === */
  --sp-2:2px;  --sp-4:4px;  --sp-6:6px;   --sp-8:8px;
  --sp-10:10px;--sp-12:12px;--sp-14:14px; --sp-16:16px;
  --sp-20:20px;--sp-24:24px;--sp-32:32px; --sp-40:40px;
}
```

## 08 迁移规则

全部为 ±1–2px 的取整,布局密度几乎不变:

| 现有值 | 迁移到 | 说明 |
|---|---|---|
| 1 / 3 / 5 / 7 / 9 / 11 / 13 / 15 | +1 进到偶数档 | 87 处,每处只动 1px(9→10、13→14) |
| 18 · 22 · 26 · 28 | 20 · 24 · 24 · 24 | 31 处,登上 4px 主网格;弹窗与内容区共用 24 节奏 |
| 2–16 偶数 · 20 · 24 | 同值换 token | 约 300 处,纯替换零视觉变化 |
| 34 / 36 / 38 / 56 / 60 / 124 | 保留组件内定义 | 图标避让、缩进、让位、场景留白等功能常量 |

### 实施顺序建议

1. `:root` 加入 `--sp-*` token(纯新增,零风险)。
2. 奇数值 +1 取整并替换为 token(87 处,每处 1px)。
3. 18 / 22 / 26 / 28 归入 20 / 24(31 处,±2px,重点回归检查内容区与弹窗)。
4. 其余偶数值同值替换为 token(机械替换)。
5. calc() 表达式内的间距分量(如底部栏让位)保持原样,只替换纯间距值。

| token | 值 | 适用 |
|---|---|---|
| `--rsm` | 4px | 小图形:封面缩略图、日历小封面、勾选框、行内 code、滑动把手指示 |
| `--rchip` | 6px | 徽章/胶囊:分类徽章、标签、筛选胶囊、计数、备注展开、展开全部 |

### 刻意保留直角(勿改)

- 手机端全屏弹窗 `.mbox`、全屏日历 `.cal-modal`(全屏页面不加圆角,平台惯例);
- 手机端透明表格容器 `.twrap`(无底无边,圆角无意义)。

### 实施注意

- 手机端笔记/备忘卡外层 `.swipe-wrap`(滑动删除容器)必须与卡片同为 `--rlg` 并保持
  `overflow:hidden`,且 `.swipe-del` 红色面板右侧两角也要 `--rlg`,否则卡片圆角处会露红边。
